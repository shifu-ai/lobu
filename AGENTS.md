## Structure
- `packages/core` — shared interfaces/utils/types. `packages/server` — embedded server + gateway (connections → `src/gateway/connections/`, orchestration → `src/gateway/orchestration/`, Slack OAuth → `src/gateway/routes/public/slack.ts`). `packages/agent-worker` — agent execution (OpenClaw, `src/openclaw/`); talks only to gateway+agent, no platform knowledge.
- **Platform isolation:** InteractionService events carry a `platform` field; each renderer filters on its own identity (`platform === "telegram"`), never another's.
- Bun workspaces under `packages/*`; TS in `packages/*/src`, tests in `__tests__`. Prefer `bun` over `npm`. Fix unused params by deleting them, not `_`-prefixing. Landing copy (`packages/landing`): no em dashes in user-facing text.
- **Submodule `packages/owletto`:** push owletto to a reachable branch BEFORE bumping the pointer (unreachable SHA breaks prod cloning). Before pushing any `bun.lock`/`package.json` change: `git submodule update --init packages/owletto && bun install --frozen-lockfile`; commit any regenerated lockfile in the same change.
- **Frontend (owletto):** before UI changes, read `packages/owletto/DESIGN_GUIDELINES.md` (not auto-loaded) and follow it; don't add new primitives without updating it in the same PR.

## Architecture
- **Platform:** all chat platforms run through Chat SDK adapters in `packages/server/src/gateway/connections/`; connections are created via the `/agents` UI or CRUD API (no per-platform env vars). Webhooks are the default transport — don't add per-platform SDKs/transports. Telegram alone has `mode: auto|webhook|polling`; `polling` is rejected when `LOBU_CLOUD_MODE=1`.
- **Orchestration:** one app process runs gateway + worker orchestrator + embeddings + memory. Workers are `child_process.spawn` subprocesses (`cwd=./workspaces/{agentId}/`, `WORKSPACE_DIR` env; Linux wraps them in `systemd-run --user --scope` for limits + `IPAddressDeny`). Postgres (pgvector; optional postgis) is the only external, via `DATABASE_URL`; runtime state lives in PG tables. Workers NEVER see real creds — the gateway `secret-proxy` swaps `lobu_secret_<uuid>` placeholders at egress.
- **🚨 Multi-replica is prod reality — every change MUST be correct under N>1 app replicas.** k8s Deployment (`charts/lobu`), `app.replicaCount`>1, `sessionAffinity: ClientIP`. Per-pod state (`SseManager` + its event backlog, the in-process `workers` map, the deploy-lock cache) is in-memory and pod-local with no cross-pod fan-out. Cross-replica delivery rides Postgres (`thread_response` queue; an event broadcast on the wrong pod is silently dropped — platform responses are owner-routed, **API/SSE are not** — a known gap). **Never put shared state in an in-memory Map/singleton another replica must read/mutate.** Before claiming a feature works, answer: *"does this hold with 3 replicas behind ClientIP affinity?"* — if it relies on one component seeing another's in-memory event across pods, it's broken; use a Postgres-mediated signal.
- **MCP:** providers from `config/providers.json`; MCP servers from per-agent settings or `SKILL.md`. Workers discover MCP tools at startup and call them via the gateway proxy (JWT). Built-ins: `AskUser`, `UploadFile`. Integration auth (OAuth/refresh) lives in Lobu MCP servers — workers never see tokens.
- **`events` is append-only.** Never `DELETE FROM events`; tombstone via `client.knowledge.delete()` / `save_knowledge({ supersedes_event_id, ... })`. `current_event_records` masks superseded rows.
- **Connectors:** `*.connector.ts` extending `ConnectorRuntime`. npm deps → project `package.json`, bundled by esbuild at compile time; native deps → `runtime.nix.packages` (nixpkgs refs), provisioned via `nix-shell` at run. Compile happens on the CLI (`lobu apply` → `bun install --ignore-scripts`); `@lobu/connector-sdk` is externalized (runtime-provided).
- **Guardrails** (`packages/core/src/guardrails/`): stages `input`/`output`/`pre-tool`; built-ins `secret-scan`, `pii-scan`, `forbidden-tools` (see `gateway/guardrails/builtins.ts` + `aggregator.ts`). Configure in `[agents.<id>].guardrails` (lobu.toml); all fail open on infra errors; each trip writes a `guardrail-trip` event.
- **Network/egress:** worker subprocesses get `HTTP_PROXY=http://localhost:8118`. Access via `WORKER_ALLOWED_DOMAINS`: empty=no net (default), `github.com`=allowlist, `*`=all (not prod), `*`+`WORKER_DISALLOWED_DOMAINS`=blocklist; domains exact or `.wildcard`. Linux prod adds kernel `IPAddressDeny=any` + allow 127.0.0.1. Risky domains can route through an LLM egress judge (Haiku, 5-min cache, fail-closed; needs `ANTHROPIC_API_KEY`; `gateway/proxy/http-proxy.ts`).

## Releasing
release-please owns it: land conventional commits on `main`, merge the generated release PR, CI publishes via OIDC (see `docs/RELEASING.md`). Inter-package deps MUST be `"@lobu/<name>": "workspace:*"`. Don't hand-edit `packages/*/package.json` versions or push `chore(release)` commits.

## Agent rules
- **Never write to `~/Code/lobu` directly — always work in a worktree.** `make task-setup NAME=<slug>` first; main stays on `main` so others can `git worktree add`. Submodule-pointer-only bump: `make bump SUBMODULE=<path> [TARGET=<ref>]`.
- Do only what's asked. Don't create `*.md` unless asked. Delete ephemeral files you create. After editing `packages/agent-worker/*`, run `make clean-workers`.
- Slack link pasted (`slack.com/archives/…?thread_ts=`) → run `scripts/slack-thread-viewer.js "<link>"` first. Unsure in planning → `codex exec "Q" --config model_reasoning_effort="high"`.
- **No new dynamic imports** outside the allow-list; default to static `import`. A new `await import(...)` needs a measured cost justification added here + a rationale comment at the call site. Allow-list: cli `index.ts`, `_lib/connector-run-cmd.ts`, `_lib/apply/apply-cmd.ts`, `_lib/apply/desired-state.ts`, `commands/dev.ts`, `memory/_lib/browser-auth-cmd.ts`; server `server.ts` (server-lifecycle), `embedded-runtime.ts` (embedded-postgres/pgvector). Tests: `await import` in `beforeAll`/`beforeEach`/`test()` is allowed (post-`vi.mock`).

## Branch hygiene
- One branch = one concern, but bundle related work (fewer, larger, reviewable PRs). Tangential ask mid-branch: commit+push current, open the PR, branch fresh off `main` (stack only if it builds on unmerged code).
- **Never `git stash`** — use WIP commits, squash later. Parallel sessions → `claude --worktree <name>`. Subagents that may switch/commit/push/destroy MUST run `isolation: "worktree"`; read-only research can share the parent.
- **Cross-repo:** owletto changes go through `make task-setup` (real branch, not detached HEAD); bump the lobu submodule pointer as a separate PR after the owletto merge. Don't pass absolute repo paths in dispatch prompts.

## Development & validation
- Prereqs: Bun, Node **22–24** (Node 25+ rejected — `isolated-vm` has no build), reachable Postgres+pgvector via `DATABASE_URL`. Optional postgis (`./scripts/setup-local-postgis.sh`) for reverse-geocoding.
- `./scripts/setup-dev.sh` (first time); `make dev` (gateway+workers+Vite on :8787); `make clean-workers` (orphans). Parallel worktrees: gitignored `.env.local` with non-default `PORT`/`WORKER_PROXY_PORT`. Don't `git switch` while a dev server runs. Husky pre-commit runs `biome check --write`.
- **After changes run `make review`** (typecheck/unit/integration + pi verdict; posts a PR comment). Build per change: landing → `cd packages/landing && bun run build`; `{core,server,agent-worker,cli}` → `make build-packages`; broad → `bun run typecheck` (`make dev` doesn't rebuild workspace pkgs).
- **E2E before merge (hard gate)** for bug fixes: red→fix→green reproducer in the PR body; if you can't reproduce, BAIL (post the dead-end, don't open a PR). Native-app UI exempt (compile-check + draft, say so).
- Bot behavior → `./scripts/test-bot.sh "@me ..."` (dev `@clawdotfreebot`, prod `@lobuaibot`; clear stale history via `chat_state_lists`). Prompt/behavior → promptfoo (`bun run evals`). Auth'd UI → `docs/BROWSER_TESTING.md`.
- **Owletto e2e (Chrome/Mac):** Chrome → `make e2e-browser` launches/reuses the stable `owletto` agent-browser harness (persistent `~/.config/lobu-dev/chrome` profile + `--extension` from this worktree); drive it with `agent-browser --session owletto <cmd>`, `RESTART=1` to pick up extension/worktree changes. The extension's fixed manifest `key` pins its ID, so chrome.storage.local (gateway URL + auth) persists — pair once, reuse across sessions/worktrees, like the installed Mac app. Mac → use the installed `Owletto.app`; it reads `~/.config/lobu/config.json` each popover so task-setup-registered worktree contexts appear in its picker. Don't reintroduce a per-worktree extension symlink.
- `.env` is the single source of truth for secrets (restart `make dev` after changes). Worker sessions persist via `./workspaces/{agentId}/`. Skills declare network/nix needs that merge into the allowlist — review skills before installing; destructive MCP calls need in-thread approval unless pre-approved in `lobu.toml`.
